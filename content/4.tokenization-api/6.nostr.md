El **Nostr Service** integra la capa social y publicitaria mediante el protocolo Nostr.

## Referencia de Endpoints

### 1. Anuncios (Kind 30402)

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `POST` | `/announcements` | Publica anuncio clasificado en Nostr. |

#### Ejemplo: Publicar Anuncio
```bash [cURL]
curl -X POST https://api.ecosystem.com/announcements \
  -H "Authorization: Bearer <token>" \
  -d '{
    "title": "Token RWA en venta",
    "content": "Descripción del activo...",
    "price_amount": 1000
  }'
```

---

### 2. Gestión de Campañas (Zaps)

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `GET` | `/campaigns` | Lista campañas propias. |
| `POST` | `/campaigns` | Crea nueva campaña de incentivos. |
| `GET` | `/campaigns/{id}` | Detalle y estadísticas de campaña. |
| `POST` | `/campaigns/{id}/fund/intraledger` | Fondeo desde balance interno. |
| `POST` | `/campaigns/{id}/fund/external` | Fondeo via Lightning externo. |
| `POST` | `/campaigns/{id}/activate` | Activa monitoreo y pagos. |
| `POST` | `/campaigns/{id}/pause` | Pausa envíos de premios. |
| `POST` | `/campaigns/{id}/cancel` | Cancela campaña y libera fondos. |

---

### 3. Auditoría y Salud

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `GET` | `/campaigns/{id}/matches` | Lista interacciones detectadas. |
| `GET` | `/campaigns/{id}/payouts` | Historial de pagos (Zaps) realizados. |
| `GET` | `/metrics` | Métricas operativas. |
| `GET` | `/health` | Estado del servicio. |
| `GET` | `/ready` | Disponibilidad. |

---

## Modelos de Datos (Parámetros)

### Crear Campaña (`POST /campaigns`)
| Campo | Tipo | Descripción |
| :--- | :--- | :--- |
| `reward_amount_sat` | integer | Monto por cada Zap. |
| `budget_total_sat` | integer | Presupuesto total. |
| `triggers` | array | Lista de disparadores (like, repost). |

::tip
Las campañas se activan mediante eventos de Nostr detectados en tiempo real.
::
