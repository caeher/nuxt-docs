El **Admin Service** es la consola de mando para operadores y administradores.

## Referencia de Endpoints

### 1. Usuarios y Roles

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `GET` | `/users` | Lista global de usuarios (Admin only). |
| `PATCH` | `/users/{user_id}` | Actualiza rol o estado de un usuario. |

#### Ejemplo: Cambiar Rol
```bash [cURL]
curl -X PATCH https://api.ecosystem.com/users/{id} \
  -H "Authorization: Bearer <token>" \
  -d '{"role": "staff", "is_staff": true}'
```

---

### 2. Tesorería y Disputas

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `POST` | `/treasury/disburse` | Desembolso de fondos manual (2FA). |
| `POST` | `/escrows/{trade_id}/resolve` | Arbitraje de disputas (2FA). |

#### Ejemplo: Resolver Disputa
```bash [cURL]
curl -X POST https://api.ecosystem.com/escrows/{id}/resolve \
  -H "Authorization: Bearer <token>" \
  -H "X-2FA-Code: 123456" \
  -d '{
    "resolution": "refund_buyer",
    "notes": "Pruebas de impago confirmadas",
    "resolution_txid": "TXID..."
  }'
```

### 3. Auditoría y Reportes

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `GET` | `/referrals/summary` | Totales plataforma de referidos. |
| `GET` | `/referrals/{user_id}` | Resumen de referidos de un usuario. |
| `GET` | `/yield/summary` | Totales plataforma de rendimientos. |
| `GET` | `/yield/{user_id}` | Rendimientos acumulados de un usuario. |

### 4. Salud y Diagnóstico

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `GET` | `/metrics` | Métricas de Prometheus. |
| `GET` | `/health` | Estado del servicio. |
| `GET` | `/ready` | Disponibilidad del servicio. |

---

## Modelos de Datos (Parámetros)

### Resolución de Disputas
| Campo | Tipo | Descripción |
| :--- | :--- | :--- |
| `resolution` | string | `refund_buyer` o `release_to_seller`. |
| `notes` | string | Justificación administrativa. |

::important
Todas las acciones financieras requieren el header `X-2FA-Code`.
::
