El **Auth Service** es el núcleo de seguridad de la plataforma. Maneja desde el registro tradicional y login vía Nostr hasta la gestión avanzada de API Keys y el flujo de cumplimiento KYC.

## Referencia de Endpoints

### 1. Gestión de Sesión y Registro

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `POST` | `/auth/register` | Crea una nueva cuenta de usuario. |
| `POST` | `/auth/login` | Valida credenciales y retorna tokens JWT. |
| `POST` | `/auth/refresh` | Refresca el token de acceso usando un refresh token. |
| `POST` | `/auth/logout` | Invalida la sesión actual del usuario. |
| `GET` | `/auth/me` | Obtiene el perfil del usuario autenticado. |

#### Ejemplo de Implementación: Login
```bash [cURL]
curl -X POST https://api.ecosystem.com/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "SecretPassword123"
  }'
```

---

### 2. Autenticación Criptográfica (Nostr)

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `POST` | `/auth/nostr/challenge` | Genera un reto para firmar con llave privada Nostr. |
| `POST` | `/auth/nostr` | Valida la firma del evento Nostr e inicia sesión. |

#### Ejemplo de Implementación: Challenge
```bash [cURL]
curl -X POST https://api.ecosystem.com/auth/nostr/challenge
```

---

### 3. Seguridad y API Keys

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `POST` | `/auth/2fa/enable` | Inicia configuración de TOTP (QR y Secreto). |
| `POST` | `/auth/2fa/verify` | Valida y activa el 2FA definitivamente. |
| `GET` | `/auth/api-keys` | Lista todas las llaves de API del usuario. |
| `POST` | `/auth/api-keys` | Genera una nueva llave de API persistente. |
| `DELETE` | `/auth/api-keys/{id}` | Revoca una llave de API específica. |
| `PATCH` | `/auth/api-keys/{id}/rotate` | Rota una llave (genera nueva, invalida vieja). |
| `POST` | `/internal/api-keys/verify` | Uso interno: verifica validez de una llave. |

---

### 4. Roles y Cumplimiento (KYC)

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `GET` | `/auth/roles/user` | Verifica si el usuario tiene rol básico. |
| `GET` | `/auth/roles/seller` | Verifica permisos de vendedor. |
| `GET` | `/auth/roles/admin` | Verifica permisos de administrador. |
| `GET` | `/auth/roles/auditor` | Verifica permisos de auditor. |
| `POST` | `/auth/kyc/submit` | Envía documentos de identidad para validación. |
| `GET` | `/auth/kyc/status` | Consulta el estado de la verificación KYC. |
| `GET` | `/auth/kyc/admin` | Lista registros KYC (Solo Admin). |
| `GET` | `/auth/onboarding/summary` | Resumen de pasos pendientes para operar. |

### 5. Utilidades y Métricas

## Modelos de Datos (Parámetros)

### Registro (`POST /auth/register`)
| Campo | Tipo | Descripción |
| :--- | :--- | :--- |
| `email` | string | Correo único del usuario. |
| `password` | string | Contraseña (mín. 8 caracteres). |
| `display_name` | string | Nombre público opcional. |

### API Keys (`POST /auth/api-keys`)
| Campo | Tipo | Descripción |
| :--- | :--- | :--- |
| `label` | string | Identificador para la llave. |
| `scopes` | array | Lista de permisos (ej: `read:wallet`). |
| `expires_at` | datetime | Fecha de caducidad opcional. |

::important
Todas las peticiones protegidas deben incluir el header `Authorization: Bearer <JWT>`.
::
