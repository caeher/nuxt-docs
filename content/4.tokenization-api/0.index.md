### Seguridad y Autenticación
La plataforma utiliza un modelo de seguridad por capas para proteger los activos de los usuarios.

::accordion
  :::accordion-item{label="Autenticación Bearer (JWT)" icon="i-lucide-key"}
  La mayoría de los endpoints protegidos requieren un token de acceso enviado en el header:
  `Authorization: Bearer <access_token>`
  :::

  :::accordion-item{label="API Keys Programáticas" icon="i-lucide-code-xml"}
  Para integraciones servidor-a-servidor, se pueden generar llaves persistentes enviadas en el header:
  `X-API-Key: <your_api_key>`
  :::

  :::accordion-item{label="Segundo Factor (2FA/TOTP)" icon="i-lucide-shield-alert" color="warning"}
  Operaciones sensibles (retiros, desembolsos) requieren un código temporal de 6 dígitos:
  `X-2FA-Code: 123456`
  :::
::

## Guía de Desarrollo

### Entornos de Red
::accordion
  :::accordion-item{label="Liquid Network" icon="i-lucide-link"}
  Capa secundaria de Bitcoin optimizada para la emisión de activos RWA, ofreciendo transacciones confidenciales y swaps atómicos.
  :::
  :::accordion-item{label="Nostr" icon="i-lucide-radio"}
  Protocolo de comunicación descentralizado usado para el descubrimiento de ofertas y autenticación.
  :::
::

### Errores Comunes
::card-group
  :::card{title="401 Unauthorized" icon="i-lucide-shield-off"}
  #description
  Token expirado o firma inválida.
  :::

  :::card{title="403 Forbidden" icon="i-lucide-lock"}
  #description
  Falta de permisos (e.g. requiere rol `admin`).
  :::

  :::card{title="422 Unprocessable" icon="i-lucide-alert-circle"}
  #description
  Error de validación en los parámetros.
  :::
::

::tip
Todos los servicios exponen `/health`, `/ready` y `/metrics` para monitoreo operativo.
::
