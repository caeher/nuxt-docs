El **Tokenization Service** gestiona el ciclo de vida completo de un activo: desde la carga de documentos legales hasta su fragmentación en la red **Liquid**.

## Referencia de Endpoints

### 1. Gestión de Activos

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `GET` | `/assets` | Lista activos filtrados por estado/categoría. |
| `POST` | `/assets` | Registra metadatos de un activo (sin archivo). |
| `POST` | `/assets/upload` | Sube archivo PDF y crea activo (Multipart). |
| `GET` | `/assets/{id}` | Obtiene detalle, AI score y datos del token. |
| `GET` | `/assets/{id}/document` | Descarga el documento PDF legal asociado. |

#### Ejemplo: Subida de Activo
```bash [cURL]
curl -X POST https://api.ecosystem.com/assets/upload \
  -H "Authorization: Bearer <token>" \
  -F "file=@escritura.pdf" \
  -F "name=Activo Inmobiliario" \
  -F "category=real_estate" \
  -F "valuation_sat=1000000"
```

---

### 2. Ciclo de Vida y Emisión

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `POST` | `/assets/{id}/evaluate` | Inicia el análisis asíncrono de IA. |
| `POST` | `/assets/{id}/tokenize` | Emite los tokens en Liquid (Requiere aprobación). |

#### Ejemplo: Tokenización
```bash [cURL]
curl -X POST https://api.ecosystem.com/assets/{id}/tokenize \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "ticker": "RWA1",
    "total_supply": 1000,
    "unit_price_sat": 100000
  }'
```

---

### 3. Utilidades y Salud

| Método | Endpoint | Descripción |
| :--- | :--- | :--- |
| `GET` | `/liquid/info` | Información del nodo y estado de Liquid. |
| `GET` | `/liquid/issuances` | Historial de emisiones realizadas. |
| `GET` | `/metrics` | Métricas operativas (Prometheus). |
| `GET` | `/health` | Estado de salud (Liveness). |
| `GET` | `/ready` | Estado de preparación (Readiness). |

---

## Modelos de Datos (Parámetros)

### Creación (`POST /assets/upload`)
| Campo | Tipo | Descripción |
| :--- | :--- | :--- |
| `file` | file | Documento legal en formato PDF. |
| `category` | string | `real_estate`, `commodity`, `invoice`, `art`. |
| `valuation_sat` | integer | Valor total estimado en satoshis. |

### Tokenización (`POST /assets/{id}/tokenize`)
| Campo | Tipo | Descripción |
| :--- | :--- | :--- |
| `ticker` | string | Símbolo (3-5 chars) para la red Liquid. |
| `total_supply` | integer | Cantidad de fracciones a crear. |

::important
La subida de archivos requiere el header `Content-Type: multipart/form-data`.
::
