# Uso

## Editar contenido

La mayor parte del sitio se mantiene editando archivos markdown dentro de `content/`.

- `content/index.md`: portada del portal.
- `content/1.getting-started/`: onboarding para quienes mantienen la documentacion.
- `content/2.tokenization-api/`: referencia API por servicio.

## Editar documentacion API

Cuando cambie un servicio, sigue este orden:

1. Confirma el cambio en el OpenAPI JSON correspondiente.
2. Actualiza la pagina markdown del servicio.
3. Ajusta el overview si cambia el conteo de operaciones o los flujos.
4. Verifica ejemplos Python y TypeScript.

## Estandares de esta documentacion

- Los nombres de placeholders deben coincidir exactamente con el spec.
- No usar placeholders genericos como `{id}` si el JSON define `{asset_id}` o `{campaign_id}`.
- Documentar auth requerida de forma explicita:
  - `Authorization: Bearer <access_token>`
  - `X-API-Key` cuando el spec lo acepte
  - `X-2FA-Code` en operaciones sensibles
- Incluir respuestas de exito y errores frecuentes.
- Mantener snippets minimos, sintacticamente correctos y consistentes.

## Donde tocar UI y configuracion

| Archivo | Uso |
| :--- | :--- |
| `app/app.config.ts` | Header, footer, toc y ajustes UI. |
| `nuxt.config.ts` | Modulos, content, llms, nitro y configuracion global. |
| `app/components/` | Componentes compartidos del sitio. |
| `app/assets/css/main.css` | Estilos globales. |

## Checklist antes de cerrar un cambio

1. La pagina renderiza bien en local.
2. No hay enlaces internos rotos.
3. No quedan rutas obsoletas o inventadas.
4. Los codigos de respuesta coinciden con OpenAPI.
5. Los snippets siguen el formato del resto del portal.

```bash [Terminal]
pnpm lint
pnpm typecheck
```

## Cambios fuera de `content/`

Si necesitas ajustar branding, navegacion o experiencia del portal, normalmente los puntos de entrada son:

- `content/index.md`
- `app/app.config.ts`
- `app/components/AppHeader.vue`
- `app/components/AppFooter.vue`

::tip
Para cambios de API, evita editar a ciegas. Abre primero el JSON en `tokenization-openapi-json/` y usa esa definicion como base.
::
