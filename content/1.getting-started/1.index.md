# Introduccion

Este proyecto no es una plantilla generica de Nuxt UI. Es el portal de documentacion de **Tokenization Ecosystem**, enfocado en la referencia API y en las guias tecnicas para servicios de tokenizacion, wallet, marketplace, admin, auth y Nostr.

## Que incluye este repositorio

- Un sitio de documentacion construido con `Nuxt 4`, `@nuxt/ui` y `@nuxt/content`.
- Contenido versionado en `content/`.
- Especificaciones OpenAPI JSON en `tokenization-openapi-json/`.
- Rutas de servidor para exponer markdown crudo y herramientas MCP en `server/`.

## Estructura clave

| Ruta | Proposito |
| :--- | :--- |
| `content/1.getting-started/` | Guia inicial para contribuidores y usuarios internos del portal. |
| `content/2.tokenization-api/` | Referencia API derivada de los OpenAPI JSON. |
| `tokenization-openapi-json/` | Fuente unica de verdad para rutas, params y respuestas de la API. |
| `app/` | Layouts, componentes y configuracion visual del portal. |
| `server/` | Endpoints auxiliares y herramientas para consumir el contenido. |

## Fuente de verdad para la API

La documentacion API debe alinearse con estos archivos:

- `tokenization-openapi-json/auth.json`
- `tokenization-openapi-json/tokenization.json`
- `tokenization-openapi-json/marketplace.json`
- `tokenization-openapi-json/wallet.json`
- `tokenization-openapi-json/admin.json`
- `tokenization-openapi-json/nostr.json`

Si una pagina markdown contradice esos JSON, el JSON gana.

## Flujo recomendado

1. Actualiza primero el OpenAPI si cambio la API.
2. Refleja el cambio en `content/2.tokenization-api/`.
3. Verifica placeholders exactos como `{asset_id}`, `{trade_id}`, `{campaign_id}` y `{user_id}`.
4. Mantén ejemplos en Python con `requests` y TypeScript con `fetch`.
5. Revisa el sitio en local antes de publicar.

::tip
La seccion `Getting Started` explica como trabajar con este portal. La referencia funcional de endpoints vive en `/tokenization-api`.
::
