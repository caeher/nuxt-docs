## Detalle por endpoint

### Registro (`POST /auth/register`)

- Proposito: crear un usuario nuevo y devolver `user` + `tokens`.
- Auth requerida: publica.
- Body JSON: `email`, `password`, `display_name`, `referrer_code`.
- Respuesta esperada: `201` con `user` y `tokens`; `422` si el payload no valida.

```python [Python]
import requests

response = requests.post(
    "https://api.plan-b2b-flow.cuboplus.caeher.com/v1/auth/register",
    json={
        "email": "user@example.com",
        "password": "SecretPassword123",
        "display_name": "Alice",
        "referrer_code": None,
    },
    timeout=30,
)
print(response.json())
```

```ts [TypeScript]
const response = await fetch('https://api.plan-b2b-flow.cuboplus.caeher.com/v1/auth/register', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'SecretPassword123',
    display_name: 'Alice',
    referrer_code: null
  })
})

console.log(await response.json())
```

- Auth requerida: publica.
- Body JSON: `email`, `password`.

print(response.json())

### Nostr (`POST /auth/nostr/challenge`, `POST /auth/nostr`)

- Proposito: iniciar sesion o registro con identidad Nostr.
- Auth requerida: publica.
- `POST /auth/nostr/challenge`: sin body, responde `challenge`, `nonce`, `kind`, `expires_in`.
- `POST /auth/nostr`: body `pubkey`, `signed_event`; responde `200` con `user` y `tokens`.

### Tokens de sesion (`POST /auth/refresh`, `POST /auth/logout`)

- Proposito: rotar o revocar refresh token.
- Auth requerida: publica segun spec.
- Body JSON en ambos endpoints: `refresh_token`.
- Respuestas: `200`; `422` en payload invalido.

### Perfil y referidos (`GET /auth/me`, `GET /auth/referrals/summary`)

- Proposito: obtener perfil autenticado y resumen de referidos.
- Auth requerida: `Authorization: Bearer <access_token>`.
- Parametros: sin path ni query.
- Respuestas: `200`.

### 2FA (`POST /auth/2fa/enable`, `POST /auth/2fa/verify`)

- Proposito: activar TOTP para operaciones sensibles.
- Auth requerida: Bearer.
- `POST /auth/2fa/enable`: sin body; responde `totp_uri` y `backup_codes`.
- `POST /auth/2fa/verify`: body `totp_code`; responde `200`.

### API Keys (`GET /auth/api-keys`, `POST /auth/api-keys`, `DELETE /auth/api-keys/{key_id}`, `PATCH /auth/api-keys/{key_id}/rotate`)

- Proposito: administrar llaves programaticas del usuario.
- Auth requerida: Bearer.
- Headers: `Authorization`; no se define `X-API-Key` aqui.
- Parametros:
  - `GET /auth/api-keys`: query opcional `user_id`.
  - `DELETE /auth/api-keys/{key_id}`: path `key_id`, query opcional `user_id`.
  - `PATCH /auth/api-keys/{key_id}/rotate`: path `key_id`, query opcional `user_id`.
- Body de creacion: `name`, `scopes`, `expires_at`.
- Respuestas:
  - `GET`: `200` con `keys`.
  - `POST`: `201` con `id`, `name`, `key`, `key_prefix`, `scopes`, `expires_at`, `created_at`.
  - `DELETE`: `200` con `message`.
  - `PATCH`: `200` con nueva llave rotada.

```python [Python]
import requests

headers = {"Authorization": "Bearer <access_token>"}

create_response = requests.post(
    "https://api.plan-b2b-flow.cuboplus.caeher.com/v1/auth/api-keys",
    headers=headers,
    json={"name": "server-prod", "scopes": ["wallet:read"], "expires_at": None},
    timeout=30,
)
print(create_response.json())
```

```ts [TypeScript]
const createResponse = await fetch('https://api.plan-b2b-flow.cuboplus.caeher.com/v1/auth/api-keys', {
  method: 'POST',
  headers: {
    Authorization: 'Bearer <access_token>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'server-prod',
    scopes: ['wallet:read'],
    expires_at: null
  })
})

console.log(await createResponse.json())
```

### Verificacion interna (`POST /internal/api-keys/verify`)

- Proposito: validacion de llaves para servicios internos.
- Auth requerida: publica segun spec.
- Body JSON: `api_key`.
- Respuesta: `200` con `valid`, `user_id`, `scopes`, `key_id`.

### Roles (`GET /auth/roles/user`, `GET /auth/roles/seller`, `GET /auth/roles/admin`, `GET /auth/roles/auditor`)

- Proposito: verificar acceso por rol.
- Auth requerida: Bearer.
- Respuesta: `200` con `status`, `actor_role`, `required_roles`.

### Onboarding (`GET /auth/onboarding/summary`)

- Proposito: ver requisitos de custody y fiat handoff.
- Auth requerida: Bearer.
- Respuesta: `200` con `user`, `kyc_status`, `custody`, `fiat_onramp_providers`, `compliance_notices`.

### KYC (`POST /auth/kyc/submit`, `GET /auth/kyc/status`, `GET /auth/kyc/admin`, `PUT /auth/kyc/admin/{user_id}`)

- Proposito: enviar, consultar, listar y actualizar estados KYC.
- Auth requerida: Bearer.
- Parametros:
  - `GET /auth/kyc/admin`: query opcional `status_filter`.
  - `PUT /auth/kyc/admin/{user_id}`: path `user_id`.
- Body:
  - `POST /auth/kyc/submit`: `document_url`, `notes`.
  - `PUT /auth/kyc/admin/{user_id}`: `status`, `rejection_reason`, `notes`.
- Respuestas:
  - `POST /auth/kyc/submit`: `201` con `kyc`.
  - `GET /auth/kyc/status`: `200` con `kyc`.
  - `GET /auth/kyc/admin`: `200` con `records`.
  - `PUT /auth/kyc/admin/{user_id}`: `200` con `kyc`.

```python [Python]
import requests

headers = {"Authorization": "Bearer <admin_access_token>", "Content-Type": "application/json"}

response = requests.put(
    "https://api.plan-b2b-flow.cuboplus.caeher.com/v1/auth/kyc/admin/usr_123",
    headers=headers,
    json={
        "status": "approved",
        "rejection_reason": None,
        "notes": "Verified against provided documents",
    },
    timeout=30,
)
print(response.json())
```

```ts [TypeScript]
const response = await fetch('https://api.plan-b2b-flow.cuboplus.caeher.com/v1/auth/kyc/admin/usr_123', {
  method: 'PUT',
  headers: {
    Authorization: 'Bearer <admin_access_token>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    status: 'approved',
    rejection_reason: null,
    notes: 'Verified against provided documents'
  })
})

console.log(await response.json())
```
