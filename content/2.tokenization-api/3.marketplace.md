# Marketplace Service

Spec fuente: `tokenization-openapi-json/marketplace.json` (`info.version = 0.1.0`).

## Detalle por endpoint

### Crear orden (`POST /orders`)

- Proposito: colocar una orden de compra o venta.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Body JSON: `token_id`, `side`, `order_type`, `quantity`, `price_sat`, `trigger_price_sat`.
- Respuesta: `201` con `order`; `422` si el payload no valida.

```python [Python]
import requests

response = requests.post(
    "https://api.ecosystem.com/orders",
    headers={
        "Authorization": "Bearer <access_token>",
        "Content-Type": "application/json",
    },
    json={
        "token_id": "tok_123",
        "side": "buy",
        "order_type": "limit",
        "quantity": 10,
        "price_sat": 50000,
        "trigger_price_sat": None,
    },
    timeout=30,
)
print(response.json())
```

```ts [TypeScript]
const response = await fetch('https://api.ecosystem.com/orders', {
  method: 'POST',
  headers: {
    Authorization: 'Bearer <access_token>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    token_id: 'tok_123',
    side: 'buy',
    order_type: 'limit',
    quantity: 10,
    price_sat: 50000,
    trigger_price_sat: null
  })
})

console.log(await response.json())
```

### Listar ordenes (`GET /orders`)

- Proposito: listar ordenes propias.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Query: `token_id`, `side`, `status`, `cursor`, `limit`.
- Respuesta: `200` con `orders` y `next_cursor`.

### Orderbook (`GET /orderbook/{token_id}`)

- Proposito: ver bids y asks de un token.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Path: `token_id`.
- Respuesta: `200` con `token_id`, `bids`, `asks`, `last_trade_price_sat`, `volume_24h`.

### Cancelar orden (`DELETE /orders/{order_id}`)

- Proposito: cancelar una orden abierta.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Path: `order_id`.
- Respuesta: `200` con `order`.

### Escrow (`GET /escrows/{trade_id}`)

- Proposito: consultar el estado de un escrow por trade.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Path: `trade_id`.
- Respuesta: `200` con `escrow`.

### Firmar escrow (`POST /escrows/{trade_id}/sign`)

- Proposito: firmar liberacion o actualizacion del escrow.
- Auth requerida: Bearer; header opcional `X-API-Key`; header opcional `X-2FA-Code`.
- Path: `trade_id`.
- Body JSON: `pset`.
- Respuesta: `200` con `escrow`; `422` en validacion fallida.

```python [Python]
import requests

response = requests.post(
    "https://api.ecosystem.com/escrows/trade_123/sign",
    headers={
        "Authorization": "Bearer <access_token>",
        "X-2FA-Code": "123456",
        "Content-Type": "application/json",
    },
    json={"pset": "cHNldP8BA..."},
    timeout=30,
)
print(response.json())
```

```ts [TypeScript]
const response = await fetch('https://api.ecosystem.com/escrows/trade_123/sign', {
  method: 'POST',
  headers: {
    Authorization: 'Bearer <access_token>',
    'X-2FA-Code': '123456',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ pset: 'cHNldP8BA...' })
})

console.log(await response.json())
```

### Firma externa (`POST /escrows/{trade_id}/external-sign`)

- Proposito: enviar una firma externa del escrow.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Path: `trade_id`.
- Body JSON: `pset`, `signer_role`.
- Respuesta: `200` con `escrow`.

### Historial de trades (`GET /trades`)

- Proposito: listar historial de trades.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Query: `token_id`, `cursor`, `limit`.
- Respuesta: `200` con `trades` y `next_cursor`.

### Abrir disputa (`POST /trades/{trade_id}/dispute`)

- Proposito: abrir una disputa sobre un trade.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Path: `trade_id`.
- Body JSON: `reason`.
- Respuesta: `201` con `dispute`.

### Resolver disputa (`POST /trades/{trade_id}/dispute/resolve`)

- Proposito: cerrar una disputa por acuerdo.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Path: `trade_id`.
- Body JSON: `resolution`, `pset`.
- Respuesta: `200` con `dispute`.

### Salud (`GET /metrics`, `GET /health`, `GET /ready`)

- Proposito: metricas, liveness y readiness.
- Auth requerida: publica.
- Respuesta: `200`.
