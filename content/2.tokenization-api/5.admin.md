## Detalle por endpoint

### Usuarios (`GET /users`, `PATCH /users/{user_id}`)

- Proposito: listar usuarios y actualizar rol.
- Auth requerida: Bearer.
- Parametros:
  - `GET /users`: query `role`, `cursor`, `limit`.
  - `PATCH /users/{user_id}`: path `user_id`.
- Body de actualizacion: `role`.
- Respuestas:
  - `GET`: `200` con `users`, `next_cursor`.
  - `PATCH`: `200` con `id`, `email`, `display_name`, `role`, `created_at`.

```python [Python]
import requests

response = requests.patch(
    "https://api.plan-b2b-flow.cuboplus.caeher.com/v1/users/usr_123",
    headers={
        "Authorization": "Bearer <admin_access_token>",
        "Content-Type": "application/json",
    },
    json={"role": "seller"},
    timeout=30,
)
print(response.json())
```

```ts [TypeScript]
const response = await fetch('https://api.plan-b2b-flow.cuboplus.caeher.com/v1/users/usr_123', {
  method: 'PATCH',
  headers: {
    Authorization: 'Bearer <admin_access_token>',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ role: 'seller' })
})

console.log(await response.json())
```

### Tesoreria (`POST /treasury/disburse`)

- Proposito: desembolsar fondos de tesoreria.
- Auth requerida: Bearer.
- Header adicional: `X-2FA-Code` opcional segun spec.
- Body JSON: `amount_sat`, `description`.
- Respuesta: `200` con `entry`; `422` si el payload no valida.

### Resolver escrow (`POST /escrows/{trade_id}/resolve`)

- Proposito: resolver una disputa de escrow como operador admin.
- Auth requerida: Bearer.
- Header adicional: `X-2FA-Code` opcional segun spec.
- Path: `trade_id`.
- Body JSON: `resolution`, `notes`, `resolution_txid`, `collected_signatures`, `settlement_metadata`.
- Respuesta: `200` con `dispute`.

```python [Python]
import requests

response = requests.post(
    "https://api.plan-b2b-flow.cuboplus.caeher.com/v1/escrows/trade_123/resolve",
    headers={
        "Authorization": "Bearer <admin_access_token>",
        "X-2FA-Code": "123456",
        "Content-Type": "application/json",
    },
    json={
        "resolution": "refund_buyer",
        "notes": "Evidence validated",
        "resolution_txid": "txid_123",
        "collected_signatures": None,
        "settlement_metadata": None,
    },
    timeout=30,
)
print(response.json())
```

```ts [TypeScript]
const response = await fetch('https://api.plan-b2b-flow.cuboplus.caeher.com/v1/escrows/trade_123/resolve', {
  method: 'POST',
  headers: {
    Authorization: 'Bearer <admin_access_token>',
    'X-2FA-Code': '123456',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    resolution: 'refund_buyer',
    notes: 'Evidence validated',
    resolution_txid: 'txid_123',
    collected_signatures: null,
    settlement_metadata: null
  })
})

console.log(await response.json())
```

### Reportes de referidos (`GET /referrals/summary`, `GET /referrals/{user_id}`)

- Proposito: ver totales globales y detalle por usuario.
- Auth requerida: Bearer.
- Path de detalle: `user_id`.
- Respuestas:
  - `GET /referrals/summary`: `200` con `referred_users`, `active_referrers`, `total_reward_sat`.
  - `GET /referrals/{user_id}`: `200` con `user_id`, `referral_code`, `referrals_count`, `total_reward_sat`, `rewards`.

### Reportes de yield (`GET /yield/summary`, `GET /yield/{user_id}`)

- Proposito: ver yield global y por usuario.
- Auth requerida: Bearer.
- Path de detalle: `user_id`.
- Respuestas:
  - `GET /yield/summary`: `200` con `users_with_yield`, `yield_tokens`, `total_yield_sat`.
  - `GET /yield/{user_id}`: `200` con `user_id`, `total_yield_sat`, `by_token`, `accruals`.

### Salud (`GET /metrics`, `GET /health`, `GET /ready`)

- Proposito: metricas, liveness y readiness.
- Auth requerida: publica.
- Respuesta: `200`.
