## Detalle por endpoint

### Resumen wallet (`GET /wallet`)

- Proposito: devolver `wallet` con saldo consolidado y contexto general.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Respuesta: `200` con `wallet`.

### Yield y custody (`GET /wallet/yield/summary`, `GET /wallet/custody`)

- Proposito: exponer resumen de yield y postura de custodia.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Respuestas:
  - `GET /wallet/yield/summary`: `200` con `yield_summary`.
  - `GET /wallet/custody`: `200` con `configured_backend`, `wallet_backend`, `signer_backend`, `state`, `withdraw_requires_2fa` y `disclaimers`.

### Fiat on-ramp (`GET /wallet/fiat/onramp/providers`, `POST /wallet/fiat/onramp/session`)

- Proposito: descubrir proveedores y abrir una sesion externa de compra.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Body de sesion: `provider_id`, `fiat_currency`, `fiat_amount`, `country_code`, `return_url`, `cancel_url`.
- Respuestas:
  - `GET`: `200` con `providers`, `compliance_notices`.
  - `POST`: `201` con `session_id`, `handoff_url`, `deposit_address`, `expires_at`, `disclaimer`.

### Lightning invoice (`POST /lightning/invoices`)

- Proposito: crear una invoice BOLT11.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Body JSON: `amount_sats`, `memo`.
- Respuesta: `200` con `payment_request`, `payment_hash`, `r_hash`, `status`, `created_at`.

### Lightning payment (`POST /lightning/payments`)

- Proposito: pagar una invoice BOLT11.
- Auth requerida: Bearer; header opcional `X-API-Key`; header opcional `X-2FA-Code`.
- Body JSON: `payment_request`.
- Respuesta: `200` con `payment_hash`, `payment_preimage`, `status`, `fee_sats`, `failure_reason`, `created_at`.

```python [Python]
import requests

response = requests.post(
    "https://api.ecosystem.com/lightning/payments",
    headers={
        "Authorization": "Bearer <access_token>",
        "X-2FA-Code": "123456",
        "Content-Type": "application/json",
    },
    json={"payment_request": "lnbc1..."},
    timeout=30,
)
print(response.json())
```

```ts [TypeScript]
const response = await fetch('https://api.ecosystem.com/lightning/payments', {
  method: 'POST',
  headers: {
    Authorization: 'Bearer <access_token>',
    'X-2FA-Code': '123456',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ payment_request: 'lnbc1...' })
})

console.log(await response.json())
```

### Invoice detail y decode (`GET /lightning/invoices/{r_hash}`, `POST /lightning/decode`)

- Proposito: consultar invoice creada o decodificar una invoice externa.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Path de detalle: `r_hash`.
- Body de decode: `payment_request`.
- Respuestas: `200`.

### Comisiones y QR (`GET /wallet/onchain/fees`, `GET /wallet/qr`)

- Proposito: consultar fee estimates y generar QR PNG.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Query de QR: `value` requerido, `box_size` opcional, `border` opcional.
- Respuestas:
  - `GET /wallet/onchain/fees`: `200` con niveles `low`, `medium`, `high`.
  - `GET /wallet/qr`: `200`.

### Direccion on-chain (`POST /wallet/onchain/address`)

- Proposito: crear una direccion de deposito on-chain.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Body: ninguno.
- Respuesta: `201` con `address`, `unconfidential_address`, `type`.

### Peg-in (`GET /wallet/pegin/address`, `POST /wallet/pegin/claim`)

- Proposito: obtener direccion de peg-in y reclamar el deposito confirmado en Liquid.
- Auth requerida: Bearer; header opcional `X-API-Key`.
- Body de claim: `raw_transaction`, `txout_proof`, `claim_script`.
- Respuestas:
  - `GET /wallet/pegin/address`: `200` con `mainchain_address`, `claim_script`.
  - `POST /wallet/pegin/claim`: `200` con `txid`, `status`.

```python [Python]
import requests

response = requests.post(
    "https://api.ecosystem.com/wallet/pegin/claim",
    headers={
        "Authorization": "Bearer <access_token>",
        "Content-Type": "application/json",
    },
    json={
        "raw_transaction": "0200000001...",
        "txout_proof": "00000020...",
        "claim_script": "0014...",
    },
    timeout=30,
)
print(response.json())
```

```ts [TypeScript]
const response = await fetch('https://api.ecosystem.com/wallet/pegin/claim', {
  method: 'POST',
  headers: {
    Authorization: 'Bearer <access_token>',

    raw_transaction: '0200000001...',

    claim_script: '0014...'
  })

```

### Retiro on-chain (`POST /wallet/onchain/withdraw`)

- Proposito: enviar BTC a una direccion externa.
- Auth requerida: Bearer; header opcional `X-API-Key`; header opcional `X-2FA-Code`.
- Body JSON: `address`, `amount_sat`, `fee_rate_sat_vb`.
- Respuesta: `200` con `txid`, `amount_sat`, `fee_sat`, `status`.

### Peg-out (`POST /wallet/pegout`)

- Proposito: enviar fondos desde Liquid a una direccion principal de Bitcoin.
- Auth requerida: Bearer; header opcional `X-API-Key`; header opcional `X-2FA-Code`.
- Body JSON: `mainchain_address`, `amount_sat`.
- Respuesta: `200` con `txid`, `amount_sat`, `status`.

```python [Python]
import requests

response = requests.post(
    "https://api.ecosystem.com/wallet/pegout",
    headers={
        "Authorization": "Bearer <access_token>",
        "X-2FA-Code": "123456",
        "Content-Type": "application/json",
    },
    json={"mainchain_address": "bc1qexample...", "amount_sat": 150000},
    timeout=30,
)
print(response.json())
```


- Auth requerida: Bearer; header opcional `X-API-Key`.
- Query: `cursor`, `limit`, `type`.
- Respuesta: `200` con `transactions`, `next_cursor`.

### Salud (`GET /metrics`, `GET /health`, `GET /ready`)

- Proposito: metricas, liveness y readiness.
- Auth requerida: publica.
